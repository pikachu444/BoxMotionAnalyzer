=========================================
|    컴포넌트 및 기능 명세서            |
=========================================

이 문서는 애플리케이션을 구성하는 주요 소프트웨어 컴포넌트(클래스/모듈)를 정의하고, 각 컴포넌트의 책임과 핵심 기능을 설명합니다. 이는 개발을 위한 청사진을 제공합니다.

---
### 1. `MainApp` (GUI 애플리케이션)
---
*   **책임:** 메인 GUI 창과 모든 위젯(버튼, 플롯, 텍스트 상자)을 관리하고, 직접적인 사용자 상호작용을 처리합니다. 사용자 입력에 따라 다른 컴포넌트의 동작을 조율합니다.
*   **사용 프레임워크:** **PySide6**
*   **핵심 기능/메서드:**
    *   `__init__()`: 메인 윈도우를 초기화하고, `PlotManager`와 `PipelineController`의 인스턴스를 생성합니다.
    *   `_init_ui()`: `gui_sketch.txt`에 정의된 대로 모든 GUI 위젯을 생성하고 배치합니다.
    *   `_connect_signals()`: 위젯의 이벤트(예: 버튼 클릭)를 해당 핸들러 메서드(예: `_on_load_csv_clicked`)에 연결합니다.
    *   `_on_load_csv_clicked()`:
        *   파일 대화상자를 열어 사용자가 CSV를 선택하게 합니다.
        *   `DataLoader`를 인스턴스화하고 `load_csv()`를 호출합니다.
        *   성공 시, 반환된 `RawData` DataFrame을 저장합니다.
        *   `DataLoader.get_plottable_targets()`를 호출하여 "데이터" 드롭다운 메뉴를 채웁니다.
        *   `PlotManager.draw_plot()`을 호출하여 초기 그래프를 표시합니다.
    *   `_on_run_clicked()`:
        *   `AnalysisConfig` 객체를 생성하고, 모든 GUI 입력 필드의 현재 값으로 속성을 채웁니다.
        *   `PipelineController.run_analysis()`를 호출하며, `AnalysisConfig` 객체, 저장된 `RawData`, 그리고 로그 출력을 위한 `update_log` 메서드를 콜백(callback)으로 전달합니다.
    *   `update_log(message: str)`:
        *   수신된 `message` 문자열을 GUI의 로그 출력 텍스트 상자에 추가합니다. 이 메서드는 실시간 피드백을 위해 `PipelineController`에 전달됩니다.
    *   `_on_export_clicked()`:
        *   "다른 이름으로 저장" 대화상자를 엽니다.
        *   `PipelineController`의 메서드를 호출하여 가장 최근의 분석 결과를 선택된 파일 경로에 저장합니다.

---
### 2. `DataLoader` (데이터 로딩 및 파싱)
---
*   **책임:** 복잡한 멀티-헤더 구조의 원본 CSV 파일을 읽고, 유효성을 검사하며, 분석에 용이한 표준 Pandas DataFrame으로 변환하는 모든 측면을 처리합니다.
*   **핵심 기능/메서드:**
    *   `load_csv(filepath: str) -> pd.DataFrame`:
        *   `AlignBoxInputGenbyExperiment.py`의 검증된 파싱 로직을 기반으로 파일을 읽습니다.
        *   파일을 한 줄씩 읽어 헤더 정보(Type, Name, Component)를 추출합니다.
        *   실제 데이터 라인을 순회하며, 'Rigid Body' 및 'Rigid Body Marker'의 'Position' 데이터만 필터링하여 추출합니다.
        *   추출된 데이터를 `{Name}_X`, `{Name}_Y`, `{Name}_Z` 형태의 컬럼을 가진 DataFrame으로 재구성하여 반환합니다.
    *   `get_plottable_targets(raw_data: pd.DataFrame) -> list[str]`:
        *   `RawData` DataFrame의 'Name'과 'Type' 컬럼을 스캔합니다.
        *   주요 "Rigid Body"와 모든 고유 "Marker" 이름을 식별합니다.
        *   이 이름들의 리스트를 반환합니다. 예: `["강체 중심", "Marker_F1", "Marker_T2"]`.

---
### 3. `PlotManager` (상호작용형 시각화)
---
*   **책임:** GUI의 플로팅 캔버스를 관리합니다. 그래프를 그리고 모든 사용자 상호작용(마우스 오버, 드래그)을 처리하는 책임을 가집니다.
*   **사용 라이브러리 제안:** **PyQtGraph** 또는 **Matplotlib** (PySide6와 통합). PyQtGraph는 대용량 데이터의 실시간 상호작용에 더 높은 성능을 보일 수 있습니다.
*   **핵심 기능/메서드:**
    *   `draw_plot(plot_widget: object, raw_data: pd.DataFrame, target_name: str, axis: str)`:
        *   `plot_widget`에서 이전 플롯을 지웁니다.
        *   `raw_data`를 필터링하여 지정된 `target_name`과 `axis`에 대한 시계열 데이터를 얻습니다.
        *   위젯에 새로운 라인 플롯을 렌더링합니다.
    *   `enable_interactions()`:
        *   PyQtGraph의 `LinearRegionItem`을 생성하여 그래프에 추가합니다.
        *   이 아이템의 `sigRegionChanged` 시그널을 `_on_region_changed` 내부 메서드에 연결합니다.
    *   `region_changed_signal: Signal(float, float)`:
        *   선택된 영역의 시작과 끝 시간을 외부로 보내는 커스텀 시그널입니다.
    *   `_on_region_changed()`:
        *   `LinearRegionItem`의 위치가 변경될 때마다 `region_changed_signal`을 발생(emit)시키는 내부 슬롯입니다.

---
### 4. `PipelineController` (백엔드 분석 로직)
---
*   **책임:** 전체 데이터 분석 워크플로우를 제어된 순서로 실행합니다. GUI로부터 완전히 분리되어(decoupled) 있으며, `QThread`에서 실행되는 것을 상정합니다.
*   **핵심 기능/메서드:**
    *   `log_message: Signal(str)`: 처리 단계별 로그 메시지를 GUI로 전달하는 시그널.
    *   `analysis_finished: Signal(pd.DataFrame)`: 분석 완료 시 최종 결과 DataFrame을 전달하는 시그널.
    *   `run_analysis(config: dict, raw_data: pd.DataFrame)`:
        *   파이프라인 실행을 시작하는 메인 진입점입니다.
        *   `_slice`, `_smooth`, `_calculate_kinematics` 메서드를 순차적으로 호출하며, 각 단계마다 `log_message` 시그널을 발생시킵니다.
    *   `_slice(...)`: Pandas 시간 기반 인덱싱으로 데이터를 자릅니다.
    *   `_smooth(...)`: `scipy.signal.butter`와 `filtfilt`를 사용하여 Butterworth 필터를 적용합니다. (`SmoothMarkerData.py` 로직 참고)
    *   `_calculate_kinematics(...)`: `scipy.interpolate.UnivariateSpline`을 사용하여 병진 속도를 계산하고, 코너 속도를 계산합니다. (`CalculateRigidBodyVelocitySmoother.py` 로직 참고)
    *   `get_latest_result() -> pd.DataFrame`:
        *   가장 최근에 계산된 운동학 DataFrame을 반환합니다.
    *   `export_to_csv(data: pd.DataFrame, output_path: str)`:
        *   주어진 DataFrame을 지정된 경로의 CSV 파일로 저장합니다.
