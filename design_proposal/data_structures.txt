=========================================
|      핵심 데이터 구조 설계              |
=========================================

이 문서는 Box Motion Analyzer 애플리케이션 내부의 데이터 관리를 위한 핵심 데이터 구조를 설계합니다. 이 설계는 명확성, 관심사 분리, 효율적인 데이터 조작에 중점을 둡니다.

---
### 1. `AnalysisConfig` 클래스
---

*   **목적:** 사용자가 GUI를 통해 설정하는 모든 값을 담는 단일 컨테이너 역할을 합니다. 이 객체 하나를 백엔드 `PipelineController`에 전달하여 모든 분석 조건을 명시합니다.
*   **구현 제안:** 파이썬 데이터클래스(`dataclass`) 또는 간단한 사용자 정의 클래스.
*   **필드 (속성):**
    *   `input_csv_path: str`
        *   설명: 사용자가 선택한 원본 CSV 파일의 절대 경로.
        *   예시: `"C:/data/raw/BOX_65_ISTA_FULL_RAW.csv"`
    *   `box_dimensions: tuple[float, float, float]`
        *   설명: 박스의 물리적 규격 (길이, 너비, 높이), 단위는 밀리미터(mm).
        *   예시: `(1578.0, 930.0, 142.0)`
    *   `slice_time_start: float`
        *   설명: 분석할 구간의 시작 시간 (초). 사용자가 그래프에서 선택.
        *   예시: `142.0`
    *   `slice_time_end: float`
        *   설명: 분석할 구간의 종료 시간 (초). 사용자가 그래프에서 선택.
        *   예시: `148.0`
    *   `plotting_target: str`
        *   설명: 그래프에 표시할 데이터 시리즈의 이름.
        *   예시: `"강체 중심"`, `"F_Marker_1"`
    *   `plotting_axis: str`
        *   설명: 그래프에 표시할 특정 축 ('X', 'Y', 또는 'Z').
        *   예시: `"Y"`
    *   `smoothing_method: str`
        *   설명: 사용할 스무딩 방법.
        *   예시: `"butterworth"`, `"moving_average"`
    *   `smoothing_cutoff: float`
        *   설명: Butterworth 필터의 컷오프 주파수 (Hz).
        *   예시: `10.0`
    *   `velocity_method: str`
        *   설명: 속도 계산 방법.
        *   예시: `"spline"`, `"finite_difference"`

---
### 2. 데이터 처리 흐름: 단일 DataFrame 누적 방식
---

*   **목적:** 여러 중간 데이터 객체나 파일을 생성하는 대신, 단일 `pandas.DataFrame` 객체를 사용하여 전체 분석 파이프라인의 데이터 흐름을 관리합니다. 이는 메모리 효율성을 높이고 데이터의 일관성을 유지하는 데 도움이 됩니다.
*   **구현 방식:**
    1.  **초기 상태:** `DataLoader`가 생성한 `RawData` DataFrame에서 시작합니다.
    2.  **파이프라인 실행:** `PipelineController`에 의해 호출된 각 `Analysis Module` (예: `Slicer`, `MarkerSmoother`, `PoseOptimizer` 등)은 이 DataFrame을 입력으로 받습니다.
    3.  **데이터 누적:** 각 모듈은 자신의 분석 결과를 입력 DataFrame에 **새로운 컬럼으로 추가**하거나 기존 컬럼을 수정한 뒤, 이 **수정/확장된 DataFrame을 반환**합니다.
    *   **예시:** `PoseOptimizer`는 자세 정보를 담은 `Box_Tx`, `Box_Ty` 등의 컬럼을 추가하고, `VelocityCalculator`는 속도 정보를 담은 `CoM_Vx`, `AngVel_Wx` 등의 컬럼을 잇달아 추가합니다.
    4.  **최종 결과:** 파이프라인의 마지막 단계를 거친 DataFrame이 모든 분석 결과를 포함하는 최종 결과물이 됩니다. `IntegrateVisualizationData.py`와 같은 별도의 데이터 통합 단계가 필요 없어집니다.
