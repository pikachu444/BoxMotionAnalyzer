=========================================
|      핵심 데이터 구조 설계              |
=========================================

이 문서는 Box Motion Analyzer 애플리케이션 내부의 데이터 관리를 위한 핵심 데이터 구조를 설계합니다. 이 설계는 명확성, 관심사 분리, 효율적인 데이터 조작에 중점을 둡니다.

---
### 1. 분석 설정 (Analysis Configuration)
---

*   **목적:** 사용자가 GUI를 통해 설정하는 모든 분석 조건을 담는 컨테이너 역할을 합니다. 현재 구현에서는 별도의 클래스 대신, `main_app.py`의 `run_pipeline` 메서드 내에서 동적으로 생성되는 **딕셔너리(dictionary)**를 사용합니다.
*   **핵심 키-값 쌍:**
    *   `slice_filter_by`: `'time'` 또는 `'frame'`. 현재는 `'time'`으로 고정되어 사용됩니다.
    *   `slice_start_val`: 분석할 구간의 시작 시간 (초).
    *   `slice_end_val`: 분석할 구간의 종료 시간 (초).
    *   (기타 `PoseOptimizer`나 `Smoother`에 필요한 파라미터들이 향후 추가될 수 있음)

---
### 2. 데이터 처리 흐름: 단일 DataFrame 누적 방식
---

*   **목적:** 여러 중간 데이터 객체나 파일을 생성하는 대신, 단일 `pandas.DataFrame` 객체를 사용하여 전체 분석 파이프라인의 데이터 흐름을 관리합니다. 이는 메모리 효율성을 높이고 데이터의 일관성을 유지하는 데 도움이 됩니다.
*   **구현 방식:**
    1.  **초기 상태:** `DataLoader`가 생성한 `RawData` DataFrame에서 시작합니다.
    2.  **파이프라인 실행:** `PipelineController`에 의해 호출된 각 `Analysis Module` (예: `Slicer`, `MarkerSmoother`, `PoseOptimizer` 등)은 이 DataFrame을 입력으로 받습니다.
    3.  **데이터 누적:** 각 모듈은 자신의 분석 결과를 입력 DataFrame에 **새로운 컬럼으로 추가**하거나 기존 컬럼을 수정한 뒤, 이 **수정/확장된 DataFrame을 반환**합니다.
    *   **예시:** `PoseOptimizer`는 자세 정보를 담은 `Box_Tx`, `Box_Ty` 등의 컬럼을 추가하고, `VelocityCalculator`는 속도 정보를 담은 `CoM_Vx`, `AngVel_Wx` 등의 컬럼을 잇달아 추가합니다.
    4.  **최종 결과:** 파이프라인의 마지막 단계를 거친 DataFrame이 모든 분석 결과를 포함하는 최종 결과물이 됩니다. `IntegrateVisualizationData.py`와 같은 별도의 데이터 통합 단계가 필요 없어집니다.
