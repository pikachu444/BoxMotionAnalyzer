# 소프트웨어 설계 문서 (v6 최종): Box Motion Analyzer GUI

## 1. 개요

이 문서는 스크립트 기반 분석 파이프라인을, **Matplotlib** 기반의 사용자 친화적 GUI 애플리케이션으로 전환하기 위한 최종 설계를 설명합니다. 주요 목표는 성능, 안정성, 사용성을 개선하고, 다중 데이터 비교, 스크롤, 양방향 동기화 슬라이싱 등 풍부한 사용자 경험을 제공하는 것입니다.

---

## 2. GUI 디자인 및 워크플로우

최종 GUI 레이아웃은 `gui_sketch.txt`에 정의되어 있으며, 주요 특징은 다음과 같습니다.
- **Matplotlib 기반 시각화:** 안정적인 Matplotlib Canvas와 제어용 툴바를 사용합니다.
- **팝업 방식 다중 선택:** "Select Data..." 버튼을 통해 **별도의 `DataSelectionDialog`**에서 여러 데이터를 선택하는 방식으로 UI를 깔끔하게 유지합니다.
- **양방향 동기화 슬라이싱:** "Enable Slicing" 체크박스로 기능을 제어하며, 그래프의 선택 영역과 입력란이 양방향으로 동기화됩니다.
- **정돈된 레이아웃:** `QGroupBox`을 사용하여 관련 컨트롤들을 시각적으로 그룹화하고, `QScrollArea`를 통해 접근성을 보장합니다.

*(상세 레이아웃은 `gui_sketch.txt` 참조)*

---

## 3. 핵심 데이터 구조 및 처리

- **`DataLoader` 최적화:** CSV 파일을 스트리밍 방식으로 읽고, 필요한 데이터만 **선별적으로 파싱**하여 로딩 속도와 메모리 효율을 극대화합니다.

*(상세 내용은 `data_structures.txt` 참조)*

---

## 4. 소프트웨어 컴포넌트 및 아키텍처

애플리케이션은 명확히 분리된 컴포넌트 기반 아키텍처로 구축됩니다.

- **`MainApp` (GUI):**
    - **책임:** 모든 GUI 위젯을 관리하고, `DataSelectionDialog`를 띄우는 등 사용자 입력을 받아 다른 컴포넌트에 작업을 위임합니다.

- **`DataSelectionDialog` (대화상자):**
    - **책임:** 다중 데이터 선택을 위한 UI를 제공하고, 선택 결과를 `MainApp`에 반환합니다.

- **`PlotManager` (시각화):**
    - **책임:** **Matplotlib**의 `Axes` 객체를 제어하여 다중 그래프, 범례, `SpanSelector`의 활성화/비활성화 및 위치 제어, **커서 값 표시(구현 예정)** 등을 담당합니다.

- **`PipelineController` (백엔드 분석 로직):**
    - **책임:** `run_analysis` 실행 시, "Enable Slicing" 체크박스 상태에 따라 전체 데이터 또는 잘라낸 데이터를 사용하여 분석을 수행합니다.
    - **리팩토링 계획 (예정):** 향후 **`kinematics_utils.py`** 와 **`config_keyword.py`** 를 생성하여 코드 구조를 개선할 예정입니다.

*(상세 기능 및 책임은 `component_specs.txt` 파일 참조)*
