# 소프트웨어 설계 문서 (v3 최종): Box Motion Analyzer GUI

## 1. 개요

이 문서는 스크립트 기반 분석 파이프라인을, **Matplotlib** 기반의 사용자 친화적 GUI 애플리케이션으로 전환하기 위한 최종 설계를 설명합니다. 주요 목표는 성능, 안정성, 사용성을 개선하고, 다중 데이터 비교, 스크롤, 조건부 슬라이싱 등 풍부한 사용자 경험을 제공하는 것입니다.

---

## 2. GUI 디자인 및 워크플로우

최종 GUI 레이아웃은 `gui_sketch.txt`에 정의되어 있으며, 주요 특징은 다음과 같습니다.
- **Matplotlib 기반 시각화:** 안정적인 Matplotlib Canvas와 제어용 툴바를 사용합니다.
- **다중 데이터 선택:** `QListWidget`을 통해 여러 마커를 동시에 선택하여 그래프에서 비교할 수 있습니다.
- **조건부 슬라이싱:** **"Enable Slicing" 체크박스**를 통해 사용자가 분석 구간 설정을 명시적으로 켜고 끌 수 있어, 전체 데이터 분석과 구간 분석 간의 전환이 직관적입니다.
- **스크롤 기능:** `QScrollArea`를 통해 설정 패널의 모든 옵션을 창 크기에 상관없이 접근할 수 있습니다.

워크플로우는 "데이터 로드 -> (선택) 다중 데이터 선택 -> (선택) 슬라이싱 활성화 및 구간 설정 -> 분석 실행 -> 결과 내보내기" 순으로 진행됩니다.

*(상세 레이아웃은 `gui_sketch.txt`, 상세 흐름은 `flow_chart.txt` 참조)*

---

## 3. 핵심 데이터 구조 및 처리

- **`DataLoader` 최적화:** CSV 파일을 스트리밍 방식으로 읽고, 필요한 데이터만 **선별적으로 파싱**하여 로딩 속도와 메모리 효율을 극대화합니다.

*(상세 내용은 `data_structures.txt` 참조)*

---

## 4. 소프트웨어 컴포넌트 및 아키텍처

애플리케이션은 수정된 컴포넌트 기반 아키텍처로 구축됩니다.

- **`MainApp` (GUI):**
    - **책임:** `Matplotlib Canvas`, `QListWidget`, `QScrollArea`, `QCheckBox` 등 핵심 위젯을 관리하고 사용자 입력을 처리합니다.

- **`PlotManager` (시각화):**
    - **책임:** **Matplotlib**의 `Axes` 객체를 제어하여 다중 그래프, 범례, `SpanSelector`의 표시/숨김 등을 관리합니다.

- **`DataLoader` (데이터 입출력):**
    - **책임:** 최적화된 방식으로 CSV 파일을 파싱합니다.

- **`PipelineController` (백엔드 로직):**
    - **책임:** `run_analysis` 실행 시, "Enable Slicing" 체크박스 상태에 따라 전체 데이터 또는 잘라낸 데이터를 사용하여 분석을 수행합니다.

*(상세 기능 및 책임은 `component_specs.txt` 파일 참조)*

---

## 5. 결론

이 최종 설계는 성능, 안정성, 사용자 경험을 모두 개선한 완전한 청사진을 제공합니다. Matplotlib으로의 전환과 GUI 기능 강화를 통해, 초기 기획 의도에 부합하는 고품질의 분석 도구를 개발할 수 있는 기반을 마련했습니다.
