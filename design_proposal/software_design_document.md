# 소프트웨어 설계 문서: Box Motion Analyzer GUI

## 1. 개요

이 문서는 현재 스크립트 기반의 박스 모션 분석 파이프라인을 사용이 편리한 그래픽 사용자 인터페이스(GUI) 애플리케이션으로 전환하기 위한 전체 설계를 설명합니다. 주된 목표는 프로그래머가 아닌 사용자도 쉽게 모션 캡처 데이터를 불러오고, 분석 파라미터를 정의하며, 데이터를 시각화하고, 효율적으로 결과를 추출할 수 있는 도구를 만드는 것입니다.

---

## 2. GUI 디자인 및 워크플로우

사용자 인터페이스와 워크플로우는 상단과 하단으로 나뉘는 구조를 통해, 시각화 영역을 극대화하고 컨트롤 영역을 명확히 분리하여 직관적으로 설계되었습니다.

### 2.1. GUI 스케치

시각적 레이아웃은 `gui_sketch.txt`에 정의되어 있으며, 주요 특징은 다음과 같습니다:
- **상단 영역 (시각화/로그):** 프로그램의 전체 가로 폭을 활용하는 넓은 시각화 창과 실시간 로그 창으로 구성됩니다.
- **하단 영역 (설정/실행):** 파일 불러오기, 박스 규격, 플롯 옵션, 분석 구간 설정 및 실행/내보내기 등 모든 컨트롤 기능을 통합하여 배치합니다.

*(상세 레이아웃은 `gui_sketch.txt` 파일 참조)*

### 2.2. 프로그램 흐름도

사용자 워크플로우는 다음과 같습니다:
1.  **불러오기:** 사용자가 CSV 파일을 불러옵니다.
2.  **설정:** 사용자가 박스 규격을 입력합니다.
3.  **시각화 및 슬라이싱:** 애플리케이션이 데이터를 플로팅합니다. 사용자는 그래프와 상호작용하여 분석할 시간 범위를 선택합니다. 이 과정은 **비파괴적(non-destructive)**으로, 원본 데이터를 수정하지 않고 선택된 영역을 시각적으로만 표시합니다.
4.  **실행:** 사용자가 "분석 실행" 버튼을 클릭하면, 백엔드 파이프라인이 **메모리 내에서** 선택된 구간의 데이터로 분석을 수행합니다.
5.  **내보내기:** 분석이 완료되면, 사용자는 결과 데이터를 새로운 CSV 파일로 내보낼 수 있습니다.

*(단계별 상세 프로세스는 `flow_chart.txt` 파일 참조)*

## 3. 핵심 데이터 구조

명확한 데이터 관리를 위해, 애플리케이션은 세 가지 핵심 데이터 구조를 사용합니다.

- **`AnalysisConfig`:** 사용자가 정의한 모든 설정값(예: 박스 규격, 슬라이스 시간)을 담는 클래스/데이터클래스 객체.
- **`RawData`:** 원본 CSV의 수정되지 않은 전체 데이터를 담고 있는 Pandas DataFrame.
- **`AnalysisData`:** 파이프라인 처리 과정에서 생성되는 중간 및 최종 DataFrame(`sliced`, `smoothed`, `kinematics` 등)을 담는 컨테이너(딕셔너리 또는 클래스).

*(상세 필드 설명은 `data_structures.txt` 파일 참조)*

## 4. 소프트웨어 컴포넌트 및 아키텍처

애플리케이션은 관심사 분리 및 유지보수성을 보장하기 위해 컴포넌트 기반 아키텍처로 구축됩니다.

- **GUI 프레임워크:** **PySide6**

- **`MainApp` (GUI):**
    - **책임:** UI와 사용자 입력을 관리하고, 다른 컴포넌트 호출을 조율.
    - **핵심 기능:** 사용자 클릭을 실제 동작으로 변환.

- **`DataLoader` (데이터 입출력):**
    - **책임:** CSV 파일을 읽고 유효성을 검사.
    - **핵심 기능:** `load_csv(filepath)`는 `RawData` DataFrame을 반환.

- **`PlotManager` (시각화):**
    - **책임:** 그래프를 그리고, 사용자가 마우스로 분석 구간을 선택할 수 있는 상호작용(`LinearRegionItem`)을 제공. **PySide6**와 호환되는 라이브러리(예: **PyQtGraph**) 사용.
    * **핵심 기능:** `draw_plot()`은 데이터를 렌더링하고, `enable_interactions()`는 구간 선택 기능을 활성화하며, `region_changed_signal`을 통해 선택된 범위를 `MainApp`에 알림.

- **`PipelineController` (백엔드 로직):**
    - **책임:** 분석 단계(슬라이스, 스무딩, 계산)를 순차적으로 실행. `QThread`에서 동작하여 GUI의 반응성을 보장.
    - **핵심 기능:** `run_analysis`는 전체 파이프라인을 조율. 기존 스크립트(`SmoothMarkerData.py`, `CalculateRigidBodyVelocitySmoother.py` 등)의 핵심 알고리즘(Butterworth 필터, Spline 미분 등)을 이식하여 사용. `log_message` 시그널을 통해 처리 과정을 GUI에 실시간으로 보고.

*(상세 기능 및 책임은 `component_specs.txt` 파일 참조)*

## 5. 결론

이 설계는 Box Motion Analyzer GUI 개발을 위한 완전한 청사진을 제공합니다. 시각화 공간을 극대화하고 사용성을 개선한 레이아웃을 포함하여, 명시된 모든 사용자 요구사항을 충족하는 명확하고 견고한 아키텍처를 제시합니다.
